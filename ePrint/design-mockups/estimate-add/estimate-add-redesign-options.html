Add New Estimate — design review (mockup only)

Based on estimate_stage1_new.ascx / estimates_add.aspx. No production code was changed. Open this file in a browser to compare options.

Problems with the current screen

  • Gray label bars on every row feel like a 2000s table form and waste horizontal space.
  • Two tall columns with uneven length — dates stack endlessly on the right with no grouping.
  • Required fields (*) are easy to miss; validation messages compete with dense rows.
  • “+” actions (new customer, contact, address) are tiny and disconnected from fields.
  • No page title or progress — users don’t know what step they’re on before “Next”.
  • Header/Footer/Comments are buried mid-form instead of in clear sections.

Recommended Option B keeps one screen (no wizard regression) but groups fields like modern CRM/ERP forms. Option C helps if you later split Stage 1 / Stage 2 explicitly.
